🤖 AI content automation system with multi-step decision pipeline

Stack: Python · OpenAI API (GPT-4o-mini) · Telegram Bot API · Twitter API v2 · Tweepy · python-dotenv

Problem - Solution

Writing quality Twitter threads manually takes 30-60 min and requires constant context-switching. Instead of a simple single-prompt bot, I built a 6-step pipeline where each step has one responsibility - idea generation, LLM filtering, thread generation, self-scoring, user selection, and auto-publishing via API. Built modular: prompt in system_prompt.txt, secrets in .env, memory in memory.json.

Pipeline

  1. Idea generation - GPT-4o-mini generates 5 raw ideas
  2. Selection - second LLM call picks 3 strongest ideas
  3. Thread generation - 3 threads x 4-7 tweets as structured JSON
  4. Self-scoring - LLM scores 1-10 and re-ranks by quality
  5. User selects 1, 2, or 3 - or requests 3 new options
  6. Auto-publish - Tweepy posts thread to Twitter via API v2 with retry logic

Results

  • Content creation time: 30-60 min → under 2 min
  • Live at @ParabolaHuman - 13 posts, 4 full threads published
  • Built in 3-4 weeks while learning Python and APIs from scratch
